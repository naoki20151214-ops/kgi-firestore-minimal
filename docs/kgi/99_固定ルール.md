固定ルール  
次のKGIの扱いルール  
Version 0.1.0  
最終更新日: 2026-04-03  
  
1. KGIは、その期限の中で完結して判定できる成果状態とする。  
  
2. 1つのKGIに、複数の時間軸を混ぜない。  
例:  
- サイトを完成して公開する  
- 公開後に収益を出す  
は、原則として別KGIに分ける。  
  
3. 今回のKGIの達成後にしか判定できない内容は、同じKGIに入れない。  
例:  
- 公開後の流入確認  
- 公開後の初回収益確認  
- 公開後のクリック確認  
- 数週間後の反応確認  
  
4. ただし、次のKGIで必要になる準備は、今回のKGIに含めてよい。  
例:  
- 収益導線をサイトに組み込む  
- 問い合わせ導線を設置する  
- 計測タグを入れる  
  
5. 次のKGIが今のKGIの結果に依存する場合は、今の段階では「次のKGI候補」として保存する。  
この時点では正式確定しない。  
  
6. 次のKGI候補は、今のKGI達成後に正式作成する。  
  
7. アプリ上では、  
- 今のKGI作成時に「次のKGI候補」を保持する  
- 今のKGI達成時に、その候補をもとに次のKGI作成を始める  
という流れを基本とする。  
  
8. 原則:  
未来の方向性は今決めてよい。  
ただし、未来の達成条件は現実を見てから確定する。
